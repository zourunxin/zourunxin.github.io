宿舍管理系统复盘

Created2021-02-02|Updated2021-02-22

整体介绍

项目大体结构为：Dao、Dao-Impl层；Service、Service-Impl层；Servlet层；entity类(JavaBean)；filter过滤器；和Util工具类

基本功能：

  1. 登录、注册功能；
  2. 分页展示用户数据功能；
  3. 修改用户信息功能；
  4. 删除和批量删除用户数据功能；
  5. 模糊查询功能

用的不是SpringMVC框架，而是自己写的servlet，这个过程参考网上教程完成，但计划学到新的技术后持续更新优化

1. Servlet

1.1 Servlet生命周期分为三个部分：

  • 初始化阶段，调用init()方法

  • 响应客户请求阶段，调用service()方法

  • 终止阶段，调用destroy()方法

1.2 项目需要完成的Servlet

DelAdminByIdServlet

根据id来删除用户。在show.jsp中，客户端在点击删除后，jsp页面获取被删用户id并传入到该servlet中执行删除操作。该servlet根据id，调用Service中的delAdminById方法完成删除指定用户数据操作。

DelAdminByCheckBoxServlet

批量删除用户。在show.jsp中，浏览器捕获用户勾选框及对应id，以数组形式传到该servlet中，该servlet依次取出id，调用Service中的delAdminById方法完成批量删除操作。循环结束后，需判定删除成功次数与传入id数是否相等，若相等，方才重定向到GetAllAdminByPageServlet中。

AdimnLoginServlet

管理员登录功能。根据login.jsp中传入的用户输入的用户名、密码表单，传入数据库判断身份是否存在

DoUpdateOrAddAdminServlet

更改信息与添加用户。servlet获取modify.jsp中传入的表单进行操作，表单数据用来构造admin对象（即Domitory对象，初始命名不规范）。根据传入表单中是否含有id来判断是执行更改信息或添加用户。若不含id，则为添加用户，admin对象传入Service中的addAdmin()方法，实现功能；若含id，则为更改信息，admin对象传入Service中的updateAdminById()方法，实现功能。

getAllAdminByPageAndFuzzyServlet

模糊查询功能。输入名字或班级均可实现查询功能，show.jsp获取输入的名字或班级表单，传入servlet中，servlet准备查询信息的部分sql语句，调用Service的getAdminByPageAndFuzzy()方法实现模糊查询。该方法按页返回符合条件的数据。

GetAllAdminByPageServlet

实现数据分页展示。show.jsp捕获页码（若无，则servlet中设置为1）并传入servlet，servlet设置每页展示数据条数、计算数据库数据总条数，计算总页数。将页码和每页数据条数传入Service，Service根据页码和数据条数取出指定位置的数据并返回。

GetAllAdminServlet

显示数据库所有数据。调用Service的getPersonAndClass()方法查询数据库，并返回所有的数据到show.jsp.

UpdateDormitoryServlet

获取将要修改用户的数据。show.jsp获取客户端指定修改用户的id，传入servlet，servlet根据id调用Service的getAdminById()方法获取该用户的所有数据，最后转发到modify.jsp页面。

2. jsp

jsp即前端展示部分代码，写的比较少，大概是仅仅够用，且不够美观

2.1 项目包含的jsp文件

error.jsp

错误显示页面。所有servlet中执行逻辑与计划不符的情况都将跳转到这一页面。

index.jsp

默认页面。新建项目时初始页面

login.jsp

登录页面。所有未登录请求均会经由过滤器重定向到该页面，收集用户登录数据给servlet实现登录功能。

modify.jsp

添加用户和修改信息页面。客户端收到修改（在show.jsp）或注册（在login.jsp）请求时转发到该页面，页面收集用户更改或添加的信息并传到servlet以更新数据库，实现添加新用户或更改用户信息功能。

show.jsp

主页面（展示页面）。用户登录后即跳转该页面，并展示数据库中用户的所有数据，可通过servlet实现分页展示。

succes.jsp

登录成功页面。初始登录成功页面，在show.jsp实现后就不用了。

3. Service

Service层，实现dao层不同接口（或实现类）各功能的组合运用

3.1 项目包含的Service层文件

DormitoryService和DormitoryServiceImpl

宿舍管理系统功能（管理员相关操作）的接口和接口实现类。准确来说应该是AdminService和AdminyServiceImpl,但因为用着了，改的话动的地方比较多，就不改了。DormitoryService与Dao层的DormitoryDao接口含有的方法一致。DormitoryServiceImpl通过实例化Dao层的DormitoryDaoImpl对象中的方法实现DormitoryServic接口中的方法。

ClassService和ClassServiceImpl

班级相关操作功能的接口和接口实现类。该接口ClassService仅含一个方法delclassbyname()，接口实现类ClassServiceImpl实现该方法：根据传入的用户名删除class表中相应用户的数据，在删除前需检查admin表中是否仍保留有该用户数据，若有，则输出提示并禁止删除；若无，则正常删除。

该ClassServiceImpl实现了Dao层的ClassDao和DormitoryDao两个类的对象，在对ClassDao的delclassbyname()方法进行操作前，需先到DormitoryDao的getPersonCountByName()方法中计算admin表中是否仍存有该用户数据。该过程体现出Service层的作用：将Dao层不同类的功能连接起来实现某个目的。

4. Dao

Dao层，实现对数据库表的一个个功能操作

4.1 项目包含的Dao层文件

DormitoryDao和DormitoryDaoImpl

数据库admin表相关操作的接口和实现类。接口DormitoryDao含对数据库admin表的增、删、改、查等操作的相关方法，实现类是DormitoryDaoImpl这些方法的实现。实现方法主要是sql语句、工具类、和javaBean的组合运用，实现具体功能。

ClassDao和ClassDaoImpl

数据库class表相关操作的接口和实现类。接口ClassDao仅含一个方法delclassbyname()，实现类ClassDaoImpl实现该方法：根据传入的用户名删除表class相关用户的数据。

5. Entity(JavaBean)

数据库表的可重用组件，以属性和方法的的形式表达和处理数据库表的各列数据

5.1 项目包含的entity文件

Dormitory

准确来说是Admin，因为是Admin表的实现类，是数据库的主表，show.jsp展示的数据来自于此。

PersonClass

准确来说是AdminClass，是记录管理员与所在班级关系表的实现类。

6. Util

工具类，基本的功能集中于此，如连接数据库，数据库的增删改和查询操作（需传入sql语句）

6.1 项目包含的Util文件

ComUtil

实现连接数据库的功能类。步骤分为：加载驱动、连接数据库、将数据库连接返回。

BaseDao

实现数据库的查询和增删改操作。有两个方法，分别是getDataByAny()，modifyData()，在调用ComUtil类获得数据库连接后，根据传入的sql语句实现相应的功能。

7. Filter

过滤器，所有未登录请求均跳转至登录页面

7.1 项目包含的Filter文件

EncodeFilter

编码过滤器。客户端的每次请求均会经过该过滤器，以处理数据表示或传输出现的各种乱码问题。

LoginFilter

登录过滤器。所有对后台申请处理的未登录请求（除了前往登陆页面和处理登录数据的请求）均会被该过滤器拦截，并重定向到login.jsp页面，提示用户输入数据登录，方可进行其他请求。

该过滤器与NewLoginFilter过滤器可实现相同功能，但处理逻辑不同。该过滤器是指定拦截某些页面请求，因此每次新增功能（servlet）或页面（jsp）均需添加拦截。

NewLoginFilter

同样是登录过滤器，与LoginFilter实现的功能一致。但它的处理逻辑是：拦截除前往登陆页面和处理登录数据的请求以外的所有其他请求，根据跳转源链接判断是否拦截。该过滤器在项目不断优化和膨胀的时候无需持续维护，较LoginFilter提高开发效率。
