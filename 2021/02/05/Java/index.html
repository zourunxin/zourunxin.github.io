数据结构及方法

数组——Array

方法：

  • 数组元素排序：Arrays.sort(T[] original)
  • 数组元素求和：Arrays.stream(T[] original).sum()
  • 从数组中获取最大值/最小值：Arrays.stream(nums).min()/max().getAsInt();
  • 数组元素统一赋值：Arrays.fill(T[] original, a) —— 数组original所有元素统一赋值为a，original可为一维或二维数组
  • 复制数组1：Arrays.copyOfRange(T[] original, int from, int to) —— 将一个原始的数组original，从下标from开始复制，复制到上标to（不包括上标to），生成一个新的数组（返回值）;
  • 复制数组2：System.arraycopy(T[] org1, a, T[] org2, b, k) —— 将数组org1起始索引为a（含），长度为k的元素复制（替换）到数组org2起始索引为b（含）的后续元素中。
  • 指定方式排序
  • Arrays和List的相互转化：
    1. Arrays.asList(array); —— Arrays转List
    2. list.toArray(); —— List转Array（Arrays的类型为Object）
  • List 移除特定整数的方法：list.remove((Integer) num) —— 要先将其转化为 Integer 才不会引起歧义

字符串——String

获取字符后，可利用java的Charcater类对字符进行操作

方法：

  • 遍历字符串字符：
    1. i递增通过charAt()循环遍历字符串字符；
    2. toCharArray()将字符串转化为字符数组再通过循环遍历；
    3. 通过subString()遍历
  • 替换字符串中某类字符为另一字符：
    1. string.replace(str1, str2)——其中str1为被替换字符，str2为替换字符；
    2. string.replaceAll(str1, str2)——（且支持正则表达式替换)
    3. 替换字符串中第一个匹配的字符为另一字符：string.replaceFirst(str1, str2);
  • 查找字符串是否包含某字符：
    1. string.contains(String)——括号内需为字符串形式
    2. string.indexOf(char)——括号内为字符形式，若含该字符，则返回对应下标，否则返回-1；
  • 字符串和数字（整型为例）的互相转化
    1. string.valueOf(Object value)——将value值由int类型转化为String类型
    2. Integer.parseInt(String)——将字符串string转化为int类型

StringBuilder与StringBuffer

进行添加操作时，添加的可以是字符或字符串

方法：

  • 声明：
    1. StringBuilder stack = new StringBuilder();
    2. StringBuffer stack = new StringBuffer();
  • 添加字符：stack.append(char c);
  • 指定位置添加字符：stack.insert(offset, char c)——在offset索引处添加字符c
  • 删除指定位置的字符：stack.deleteCharAt(i);
  • 转字符串格式：stack.toString();

栈——Stack

先进后出
使用栈和双端队列均可实现

方法：

  • 声明（使用栈）：Stack<Integer> stack = new Stack<>();
  • 声明（使用双端队列）：Deque<Integer> stack = new LinkedList<>();
  • 压入元素：stack.push(int value);
  • 取出栈顶元素：stack.peek();（若栈为空，则报NullPointException异常）
  • 取出栈顶元素并删除：stack.pop();

队列——Queue

先进先出

方法：

  • 声明：Queue<Integer> queue = new LinkedList<>();
  • 添加元素：queue.offer(int value);（成功则返回true，失败返回false）
  • 添加元素：queue.add(int value);（成功则返回true，失败则抛出IIIegaISlabEepeplian异常）
  • 添加元素：queue.put(int value);（无返回值，若队列满则阻塞）
  • 返回队列头部元素：queue.peek();（若队列为空，则返回null）
  • 返回队列头部元素：queue.element();（若队列为空，则抛出NoSuchElementException异常）
  • 移除并返回队列头部元素：queue.poll();（若队列为空，则返回null）
  • 移除并返回队列头部元素：queue.remove();（若队列为空，则抛出NoSuchElementException异常）
  • 移除并返回队列头部元素：queue.take();（若队列为空，则阻塞）
  • 移除队列头部元素：queue.removeFist();　　移除队列尾部元素：queue.removeLast();
  • 返回队列头部元素：queue.peekFist();　　　返回队列尾部元素：queue.peekLast();
  • 队头添加元素：queue.addFirst();　　　　 　队尾添加元素：queue.addLast();

优先级队列——PriorityQueue

优先级队列实现了queue接口，是升序队列，数组形式，左侧为队头，右侧为队尾，本质是小根堆。每次插入元素会自动重新排序，以确保队列元素保持为升序顺序。各功能同用以上普通队列方法实现（C++的优先队列每次取最大元素）
优先级队列每次以O(1)时间复杂度取出队列权值最小的元素（即队头元素），以O(log)插入新元素。

  • 声明：PriorityQueue<Integer> pq = new PriorityQueue<>();
  • 声明（大根堆）： 简化版：PriorityQueue<Integer> queue = new PriorityQueue<>((o1, o2)->o2.compareTo(o1));

Map

元素为键值对，一个map中仅能含一个键(Key)，可以在多个键中存有相同的值(value)。

方法：

  • 添加元素：map.put(Key, Value);（添加一对键值对）

  • 添加元素：map.putAll(map);(添加其他map中的键值对）

  • 获取value：map.get(Object Key);（根据key获取指定的value）

  • 删除元素：map.remove(Object Key);（根据键删除键值对）

  • 查找某键：map.containsKey(Object Key);（查找是否存在键为Key的键值对，若有，则返回true；若无，则返回false）

  • 查找某值：map.containsValue(Object Value);（查找是否存在值为Value的键值对，若有，则返回true；若无，则返回false）

  • map.getOrDefault(key, value);（若map中含有该键key，则使用该键的值；否则使用默认值value）

  • for循环遍历map中的键值对：

  • for循环遍历map中的键或值：for(Object Key : map.keySet()) 和 for(Object Value : map.values())

配置Gradle使用maven本地仓库

Java 注解

@PostConstruct 注解

从 Java EE5 规范开始，Servlet 中增加了两个影响 Servlet 生命周期的注解，@PostConstruct 和 @PreDestroy，这两个注解被用来修饰一个非静态的 void（）方法。
被 @PostConstruct 修饰的方法会在服务器加载 Servlet 的时候运行，并且只会被服务器执行一次。PostConstruct 在构造函数之后执行，init()方法之前执行。PreDestroy()方法在destroy()方法执行之后执行

setAccessible(true)用法及意义

Accessible 属性是继承自 AccessibleObject 类. 功能是启用或禁用安全检查；
setAccessible() 可作用于两处：

  1. 作用于方法上：method.setAccessible(true);
  2. 作用于属性上：field.setAccessible(true);
    将此对象的 accessible 标志设置为指示的布尔值。值为 true 则指示反射的对象在使用时应该取消 Java 语言访问检查；值为 false 则指示反射的对象应该实施 Java 语言访问检查。由于JDK的安全检查耗时较多.所以通过 setAccessible(true) 的方式关闭安全检查就可以达到提升反射速度的目的。
